# Plan de Proyecto AGENDO

## Metodología de Trabajo
Para la gestión y control del proyecto AGENDO utilizaremos metodologías ágiles. Principalmente Scrum para la división del trabajo en sprints y asignación de roles, y Kanban para el manejo visual del flujo de tareas.

## Cronograma y Fases del Proyecto

| Fase       | Duración   | Actividades principales                         |
|------------|------------|------------------------------------------------|
| Inicio     | 1 semana   | Definición de requisitos, análisis y planeación|
| Planeación | 1 semana   | Diseño, cronograma, asignación de tareas       |
| Ejecución  | 4 semanas  | Desarrollo, pruebas unitarias                    |
| Pruebas    | 2 semanas  | Pruebas de integración y corrección de errores |
| Cierre     | 1 semana   | Documentación final y entrega                     |

## Roles y Responsabilidades

- **Product Owner:** 
  Define y prioriza las funcionalidades del proyecto.

- **Scrum Master:** 
  Facilita la organización y seguimiento del proyecto.

- **Equipo de Desarrollo:**  
  Se encargan de la implementación y pruebas.

## Gestión de Tareas con Kanban
Las tareas se gestionarán utilizando un tablero Kanban digital en GitHub, con las siguientes columnas:

- Pendientes (Backlog)
- Por hacer (To Do)
- En progreso (In Progress)
- En revisión (In Review)
- Completadas (Done)

Además de este plan, en el repositorio se mantendrá actualizado el archivo README.md con la descripción general del proyecto, objetivos, tecnologías y estructura del equipo.
